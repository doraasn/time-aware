# Time Aware Skill

## 描述
当用户谈论到时间性词语（今天、明天、昨天、本周、上周、下周等）时，自动获取当前时间并基于准确时间进行分析和回答。

## 触发条件
- 用户提到：今天、明天、昨天、本周、上周、下周、本月、上月、下月、现在、当前、最近、刚才、之前、之后、未来、过去等时间性词语
- 用户询问与时间相关的问题
- 用户讨论日程、计划、截止日期等

## 工作流程

### 1. 检测时间性词语
当检测到用户消息中包含时间性词语时，立即执行时间获取。

### 2. 获取当前时间
根据设备类型使用对应的系统命令获取准确时间：

#### Windows
```powershell
Get-Date -Format "yyyy-MM-dd HH:mm:ss dddd"
```

#### macOS / Linux
```bash
date "+%Y-%m-%d %H:%M:%S %A"
```

#### Android (Termux)
```bash
date "+%Y-%m-%d %H:%M:%S %A"
```

#### iOS (iSH)
```bash
date "+%Y-%m-%d %H:%M:%S %A"
```

### 3. 时间分析
基于获取到的准确时间，分析用户的问题：
- 计算时间差（如"3天前"、"下周二"）
- 确定具体日期
- 验证用户提到的时间是否准确

### 4. 回答策略
- **有足够证据**：基于准确时间给出明确回答
- **时间不明确**：询问用户以获取更多信息，不要猜测
- **存在歧义**：列出可能的解释并请用户确认

## 示例

### 用户：今天是星期几？
**正确做法**：
1. 获取当前时间
2. 基于准确时间回答"今天是XXX"

### 用户：我昨天说过什么？
**正确做法**：
1. 获取当前时间
2. 计算"昨天"的具体日期
3. 如果没有相关记录，诚实说明

### 用户：下周有个会议
**正确做法**：
1. 获取当前时间
2. 计算下周的具体日期范围
3. 询问用户具体是哪一天

## 注意事项
- 永远不要假设当前时间
- 始终通过系统工具获取准确时间
- 如果无法获取时间，明确告知用户
- 时间相关回答必须有准确的时间依据
